### 시도
	 - 구글에서 크롤링 한 이미지 100개 중에서 59개 DB에 저장 ( 에러나는 사진 제외 )
		- 개인DB에는 들어가지만 AWS에서 생성한 공용DB에는 안들어가짐 ( 권한 문제이거나 다른 문제일 가능성 있음 )
	 ▶ 강사님의 조언을 듣고 이미지의 경로를 DB에 저장하는 방법을 찾아봄 ◀
	 	1. DB에 이미지 경로를 저장하는 방법
	 	2. 검색해보니 위치라는 것은 경로의 특성상 바뀔 소지가 다분이 있기 때문에 DB에는 파일명만 저장하고 경로명은 프로퍼티에 따로 뺴서 저장하는 방법
			- 경로가 바뀌더라도 특정 프로퍼티에 저장되어 있는 경로 값만 바꿔주면 되기 때문에 DB 업데이트가 전혀 필요 없음, 나중에 풀 경로를 만들때 프로퍼티 값 + DB값을 해주면 되기 때문


### 이슈
	- 크롤링한 사진을 DB에 넣는 도중 문제 발생
		- 크롤링한 사진 중 DB에 들어가는 사진이 있고 안들어가는 사진이 있음 
		▶ 변환 후 문자열의 길이가 길어져서 안들어가는 것으로 확인됨 
		 - 하나 밖에 저장이 안됨 ( 해결완료 ) 
	 - 공용DB에 저장하려니 자꾸 오류가 뜸 
	 	 - 이미지의 경로를 넣으면 더 효율적으로 관리 가능
